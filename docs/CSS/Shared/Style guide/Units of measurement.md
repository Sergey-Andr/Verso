В этом разделе описаны как работать с **единицами измерения** и их **комментариями**
### Единицы измерения

Используются **относительные единицы измерения**, вместо статичного `px`

Приоритетные единицы:
- **`rem`**: для размеров шрифтов, отступов, ширины и высоты.
- **`vh`, `vw`, `dvh`**: для элементов, размер которых должен зависеть от высоты или ширины экрана.

> 💡 **Лучшая практика:**
> Рядом со значением в `rem` всегда оставлять комментарий с его эквивалентом в пикселях (при базовом размере шрифта 16px).

**Пример**
```css
.main-panel {
    padding: 1.5rem; /* 24px */
}
```

### Комментарии к единицам

Все неочевидные магические числа по типу [[CSS/Mobile/Custom Prefixes#^custom-prefixes|25.8125rem]] или нестандартные решения как в [[CSS/Mobile/Custom Prefixes#^custom-prefixes|clamp]] сопровождаются комментарием в коде с пояснением причины в максимально допустимо сжатом формате

> ✍️ **Замечание**
> Магическими считаются дробные значения *не равные четверти .25*, нестандартные брейкпоинты, clamp, calc или любые значения, которые не очевидны при беглом просмотре

**пример**
```css
/* медиа-запрос с нестандартным брейкпоинтом */
@media (min-width: 25.8125rem) /* 413px; ширина, на которой ломается верстка текста */ {  
    .wsm\:leading-9 {  
        @apply leading-9;  
    }  
}

/* простая конвертация */
padding: 1.5rem; /* 24px */

/* дробное значение: нестандартный брейкпоинт */
min-width: 25.8125rem; /* 413px: фикс бага с переносом */
```

### Магические значения в JSX

Комментарии **внутри JSX** считаются **антипаттерном**.  
Вместо этого придерживаемся тех же правил, что и с константами:

1. ❗ **Если магических значений ≤ 3** — выносим их в константы **рядом с компонентом**
    
2. 📦 **Если > 3** — выносим их в отдельный файл, например `uiConstants.ts`
    

**Пример:**
```JSX
// внутри компонента
const PANEL_MARGIN_TOP = "3.125rem"; // 50px: компенсируем высоту шапки

return <div style={{ top: PANEL_MARGIN_TOP }} />;

```
